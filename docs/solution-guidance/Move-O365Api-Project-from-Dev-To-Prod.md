# <a name="deploying-development-office-365-sites-to-microsoft-azure"></a>Развертывание Office 365 сайты разработки в Microsoft Azure #

### <a name="summary"></a>Summary ###

При разработке любой тип веб-приложения, разработка для большинства выполняется локально с помощью http://localhost. Некоторые проекты использовать локальные ресурсы или сочетание локальных и удаленных ресурсов. Использование таких проектов из среды разработки локального включает в себя небольшое количество задачи, выполняемые как изменение строки подключения базы данных, URL-адреса, конфигурации, и т.д.

Веб-проектов, использующих возможности API-интерфейсы Office 365, практически не отличаются. Эти проекты используют службы Azure AD корпорации Майкрософт для проверки подлинности приложений и получения маркера доступа OAuth 2.0. Эти маркеры используются веб-приложений для проверки подлинности с помощью интерфейсов API Office 365.

На этой странице Описание действий, необходимых для создания заметок в проект разработки Office 365 API и запуска его примеры размещенных полностью в Microsoft Azure с использованием [Office 365](http://products.office.com/en-us/business/explore-office-365-for-business), [Azure Active Directory](http://azure.microsoft.com/en-us/services/active-directory/) & [Azure веб-сайтов] (http:// Azure.Microsoft.com/en-US/Services/WebSites/.

Развертывание Office 365 API веб-приложения в Microsoft Azure из среды разработки локального требуются три основных действия нужно выполнить, как описано на этой странице:

- [Создание и настройка веб-сайта Azure](#create-and-configure-an-azure-website)
- [Настройка приложения Azure AD](#configure-the-azure-ad-application)
- [Настройка проекта ASP.NET](#configure-the-aspnet-project)
- [Развертывание Office 365 API веб-приложения ASP.NET](#deploy-the-office-365-api-aspnet-web-application)

> На этой странице предполагается, что локальное приложение ASP.NET работа с использованием API Office 365. Для ссылки он будет использовать project **[O365 WebApp SingleTenant](https://github.com/SharePoint/O365-WebApp-SingleTenant)** , обнаруженных в учетной записи **[разработчик офисных решений](https://github.com/SharePoint)** в репозиториев.

# <a name="create-and-configure-an-azure-website"></a>Создание и настройка веб-сайта Azure

На этом этапе создается Azure веб-сайт, который будет использоваться для размещения веб-приложения. 

1. Перейдите к странице [Портала управления Azure](https://manage.windowsazure.com) и входа в систему с помощью учетной записи идентификатор организации.
1. После входа, в боковой панели навигации выберите **веб-САЙТОВ**.
1. На странице " **веб-сайтов** " щелкните ссылку **Создать** в нижнем колонтитуле, обнаруженных в левом нижнем углу страницы.
1. В окне мастера выберите **Быстрое создание**, введите имя для сайта в поле **URL-адрес** , выберите **Размещение планирование Web** и **подписки**. 

  ![Параметры быстрого создания: поле URL-адрес имеет значение o365api-01, Web размещения планирование задано значение Default1 (Восточный США, стандартный) подписки задано значение Azure MSDN (основной).](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-01.png)

  > Убедитесь, что хранить сообщения с именем веб-сайта, которые создаются как он понадобится позже.

1. И, наконец, щелкните ссылку **Создать веб-сайт** для создания сайта.

Предоставление Azure в некоторое время для создания сайта. После создания сайта можно указать *Параметры приложения* с помощью веб-интерфейса. Это позволяет изменить любой `<appSettings>` в рамках проекта `web.config` файл в веб-интерфейсе администрирования для веб-сайта без развертывания веб-узла базы кода для простой `web.config` изменений.

1. Выберите веб-сайт, который вы только что создали **Портала управления Azure**.
1. Щелкните ссылку **НАСТРОИТЬ** в верхней панели навигации.
1. Прокрутите список вниз до раздела **Настройки приложения** и добавления трех новых записей:
  - **IDA: ClientID**
  - **IDA: пароль** 
  - **IDA: TenantID** 
1. Скопируйте соответствующие значения из проекта рабочего `web.config` эти значения параметров веб-сайта Azure как показано на следующем рисунке:

  ![WEBSITE_NODE_DEFAULT_VERSION 0.10.32, ida: ClientID 92b1e137-c36f-4bfe-9e1c-01ef546ce4a9, ida: Password — Bns06N18ZiyYfMcyU9qUfGnZbnkBiPZfUptLDsU6cml, частично исправленный ida: TenantId. Центр номера GUID являются - 45ee - 8afc-.](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-02.png)

1. В нижнем колонтитуле нажмите кнопку **Сохранить** , чтобы сохранить изменения.

На этом этапе веб-сайта Azure — это программа установки и настройки для размещения веб-проекта Office 365 API для развертывания на более позднем этапе.

# <a name="configure-the-azure-ad-application"></a>Настройка приложения Azure AD

На этом этапе будет изменять приложения Azure AD используется для разработки и тестирования приложения Office 365.

1. Перейдите к странице [Портала управления Azure](https://manage.windowsazure.com) и входа в систему с помощью учетной записи идентификатор организации.
1. После входа, с помощью боковой панели навигации, выберите пункт **ACTIVE DIRECTORY**.
1. На странице " **active directory** " выберите каталог, который связан с клиента Office 365.
1. Выберите элемент **приложения** в верхней панели навигации.
1. В разделе **Свойства** обновление **SIGN-ON URL-адрес** для указания на URL-адрес по умолчанию созданного веб-сайта Azure. Запомните использование конечную точку HTTPS, которая предоставляется со всех веб-сайтов Azure.

  ![Имя имеет значение O365 WebApp SingleTenant.Office365App, URL-адрес входа задано значение https://o365api-01.azurewebsites.net](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-03.png)

1. В разделе **Единого входа** обновите **Приложения идентификатор URI** для использования в домен для веб-сайта Azure (как показано на следующем рисунке).
1. Затем обновите **URL-адрес ОТВЕТА** , поэтому только URL-адреса из списка домашнюю страницу веб-сайта Azure:

  ![Идентификатор URI приложения https://o365api-01.azurewebsites.net/O365-WebApp-SingleTenant, URL-адрес ответа https://o365api-01.azurewebsites.net/](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-04.png)

1. В нижнем колонтитуле нажмите кнопку **Сохранить** , чтобы сохранить изменения.

На этом этапе приложение Azure AD, используемых в Office 365 API веб-проект был настроен для работы с новой веб-сайта Azure.

# <a name="configure-the-aspnet-project"></a>Настройка проекта ASP.NET

На этом этапе вы настроите проекта ASP.NET в приложении для использования нового веб-сайта Azure.

Для примера приложения, используемые в этом примере для данное руководство дополнительные действия не требуется. Тем не менее веб-приложения, содержащие настройки в `web.config` файла для приложения Azure AD и Azure AD клиента, который используется во время разработки. Некоторые разработчики следующим шагом нужно использовать другой Azure AD приложений или даже другой Azure подписки для их разработку и производственные экземпляры.

В предыдущем шаге, описанные на этой странице, при создании веб-сайта Azure задайте параметры надстройки для приложения, которое обычно находятся в `web.config`. Чтобы убедиться в веб-приложение получает эти значения из конфигурации веб-сайта Azure, рекомендуется заменить значения в пределах `web.config` с заполнитель со значениями вместо этого.

1. Откройте проект `web.config` файла.
1. Найдите параметры надстройки для **ida: ClientID**, **Ida: пароль** и **ida: TenantId**.
1. Замените значение-заполнитель значения следующих параметров:

  ````xml
  <add key="ida:TenantId" value="set-in-azure-website-config" />
  <add key="ida:ClientID" value="set-in-azure-website-config" />
  <add key="ida:Password" value="set-in-azure-website-config" />
  ````

1. Сохраните изменения.

На этом этапе веб-приложения, веб-сайта Azure и приложения в Azure AD, все правильно настроены и готовы к развертыванию.

# <a name="deploy-the-office-365-api-aspnet-web-application"></a>Развертывание Office 365 API веб-приложения ASP.NET

На этом этапе вы публикуете веб-приложения Office 365 API для веб-сайта Azure. После развертывания сайта можно проверить его, чтобы убедиться, что все работает, при необходимости.

> На этом шаге предполагается, у вас есть он Microsoft [Azure SDK](http://azure.microsoft.com/en-us/downloads/), версии 2.0 или более поздней версии, установлены. 

## <a name="deploy-the-aspnet-web-application"></a>Развертывание веб-приложения ASP.NET

1. Откройте веб-приложения Office 365 API в Visual Studio 2013.
1. В окне Обозреватель решений средство щелкните правой кнопкой мыши проект и выберите **Опубликовать** запустить мастер **Публикация веб-сайта** .
1. На вкладке **профиля** выберите **Веб-сайт Microsoft Azure**.

  На этом этапе вам будет предложено входа в Azure подписки с помощью вашей организации по идентификатору.

1. После входа выберите веб-сайт, созданный в предыдущем шаге на этой странице и нажмите **кнопку ОК**.

  ![Диалоговое окно выберите существующий веб-сайт показан набор существующих веб-сайтов для o365api-01.](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-05.png)

1. На вкладке **подключения** нажмите кнопку **Проверить подключение** , чтобы проверить профиль подключения был успешно загружен и применения.

  ![Стрелка указывает на кнопку Проверить подключение около внизу диалогового окна с зеленая галочка рядом с кнопкой.](media/Move-O365Api-Project-from-Dev-To-Prod/Move-O365Api-Project-from-Dev-To-Prod-06.png)

1. Публикация веб-приложения и веб-сайта Azure, нажмите кнопку **Опубликовать** .

## <a name="test-the-aspnet-web-application"></a>Тестирование веб-приложения ASP.NET

После публикации веб-приложения и веб-сайта Azure, Visual Studio откройте браузер и перейдите на домашнюю страницу веб-узла. 

По умолчанию это конечной точки HTTP. При настройке приложения Azure AD, задайте его на получение дополнительные модули входа из конечную точку HTTPS отозвать из предыдущего шага. Прежде чем использовать обновления приложения URL-адрес для указания на конечную точку HTTPS.

1. В браузере обновите URL-адрес, чтобы перейти на домашнюю страницу HTTPS для веб-сайта Azure. В примере на этой странице, которая является https://o365api-01.azurewebsites.net.
1. Щелкните ссылку **Войти** в заголовок в верхней правой части страницы. Вы будете направлены на Azure AD входа на странице.

  > Если на этом этапе сообщение об ошибке, значит, вероятно проблема с тремя параметрами надстройки, созданной для веб-сайта Azure. Вернитесь и убедитесь в том, что значения являются правильными значениями из клиента Azure AD и приложения. Должно появиться URL-адрес, которая выглядит 

1. После успешного входа будет перенаправлен обратно на домашнюю страницу веб-приложения веб-сайта Azure, которую вы создали.

На этом этапе успешно развернута проекта Office 365 API веб-приложения для запуска в Azure веб-сайта.
