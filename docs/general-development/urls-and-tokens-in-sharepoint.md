## <a name="types-of-urls-in-sharepoint"></a><span data-ttu-id="4562e-104">Типы URL-адресов в SharePoint</span><span class="sxs-lookup"><span data-stu-id="4562e-104">Types of URLs in SharePoint</span></span>
<span data-ttu-id="4562e-105"><a name="TypesOfURLs"> </a></span><span class="sxs-lookup"><span data-stu-id="4562e-105"><a name="TypesOfURLs"> </a></span></span>

<span data-ttu-id="4562e-p101">SharePoint анализирует строки URL-адресов для определения их формы на основе указанного протокола (например, **http:**) или размещении косой черты (/) в строке. В зависимости от конкретного элемента можно использовать следующие формы URL-адресов:</span><span class="sxs-lookup"><span data-stu-id="4562e-p101">SharePoint parses URL strings to determine the form of URL based on a specified protocol (for example, **http:**) or on the placement of a forward slash (/) within the string. Depending on the particular member, you can use the following URL forms:</span></span>
  
    
    

- <span data-ttu-id="4562e-108">**Абсолютный URL-адрес** указывает полный путь и начинается с обозначения протокола.</span><span class="sxs-lookup"><span data-stu-id="4562e-108">An **absolute URL** specifies a full path and begins with a protocol.</span></span> <span data-ttu-id="4562e-109">Например, `http://` _домен_или_сервер_/[ `sites/`] _сайт_/ `Lists`/ _название_списка_/ `AllItems.aspx`.</span><span class="sxs-lookup"><span data-stu-id="4562e-109">For example, `http://` _domain_or_server_/[ `sites/`] _Web_Site_/ `Lists`/ _List_Title_/ `AllItems.aspx`.</span></span>
    
  
- <span data-ttu-id="4562e-110">**URL-адрес относительно домена** основан на адресе домена (он может быть именем сервера) и всегда начинается с косой черты.</span><span class="sxs-lookup"><span data-stu-id="4562e-110">A **domain-relative URL** is based on the domain (which might be the name of a server) address and always begins with a forward slash.</span></span> <span data-ttu-id="4562e-111">Он определяет полный путь от веб-сайта верхнего уровня к имени файла.</span><span class="sxs-lookup"><span data-stu-id="4562e-111">It specifies a complete path from top-level website to file name.</span></span> <span data-ttu-id="4562e-112">Например, /[`sites/`] _веб_сайт_/ `Lists`/ _название_списка_/ `AllItems.aspx`.</span><span class="sxs-lookup"><span data-stu-id="4562e-112">For example, /[ `sites/`] _Web_Site_/ `Lists`/ _List_Title_/ `AllItems.aspx`.</span></span> 
    
  
- <span data-ttu-id="4562e-113">**URL-адрес относительно веб-сайта** основан на адресе объекта веб-сайта ([SPWeb](https://msdn.microsoft.com/library/Microsoft.SharePoint.SPWeb.aspx)).</span><span class="sxs-lookup"><span data-stu-id="4562e-113">A **website-relative URL** is based on the address of a website object ( [SPWeb](https://msdn.microsoft.com/library/Microsoft.SharePoint.SPWeb.aspx) ).</span></span> <span data-ttu-id="4562e-114">Он _не_ начинается с косой черты и указывает полный путь от адреса веб-сайта к имени файла.</span><span class="sxs-lookup"><span data-stu-id="4562e-114">It does _not_ begin with a forward slash, and it specifies a complete path from the website address to the file name.</span></span> <span data-ttu-id="4562e-115">Например, `Lists/` _название_списка_/ `AllItems.aspx`.</span><span class="sxs-lookup"><span data-stu-id="4562e-115">For example, `Lists/` _List_Title_/ `AllItems.aspx`.</span></span>
    
  
- <span data-ttu-id="4562e-p105">**URL-адрес относительно файла или папки** основан на папке, содержащей файл. Он _не содержит_ косых черт и просто указывает имя файла. Например, `AllItems.aspx`.</span><span class="sxs-lookup"><span data-stu-id="4562e-p105">A **URL relative to a file or folder** is based on the folder containing the file. It does not contain _any_ forward slashes. It simply specifies the name of the file. For example, `AllItems.aspx`.</span></span>
    
> [!NOTE] 
> <span data-ttu-id="4562e-120">Не существует такого понятия, как "URL-адрес относительно семейства веб-сайтов". Передача такого URL-адреса может вызвать сбой кода.</span><span class="sxs-lookup"><span data-stu-id="4562e-120">There is no concept of a "site collection–relative URL"; passing such a URL may cause the code to fail.</span></span> 
  
    
    


## <a name="good-practice-for-image-urls"></a><span data-ttu-id="4562e-121">Рекомендации для URL-адресов изображений</span><span class="sxs-lookup"><span data-stu-id="4562e-121">Good practice for image URLs</span></span>
<span data-ttu-id="4562e-122"><a name="GoodPracticeImageURL"> </a></span><span class="sxs-lookup"><span data-stu-id="4562e-122"><a name="GoodPracticeImageURL"> </a></span></span>

<span data-ttu-id="4562e-p106">При создании URL-адреса файла изображения, расположенного в каталоге %ProgramFiles%\\Common Files\\Microsoft Shared\\web server extensions\\15\\TEMPLATE\\LAYOUTS\\1033\\IMAGES, укажите путь, использующий корневой веб-сайт семейства сайтов, но не включающий дочерний сайт. Например, используйте путь /_layouts/images/MyImage.gif вместо /MySubsite/_layouts/images/MyImage.gif. Это связано с тем, что URL-адреса дочерних сайтов сопоставляются по-разному в зависимости от того, где они используются. Эти варианты можно игнорировать, если вы всегда используете URL-адрес относительно корневого веб-сайта.</span><span class="sxs-lookup"><span data-stu-id="4562e-p106">When you create a URL to an image file that is located in the %ProgramFiles%\\Common Files\\Microsoft Shared\\web server extensions\\15\\TEMPLATE\\LAYOUTS\\1033\\IMAGES directory, specify a path that uses the root website of the site collection, but that does not include a subsite within the path. For example, use /_layouts/images/MyImage.gif for an image file, not /MySubsite/_layouts/images/MyImage.gif. This is because subsite URLs are resolved in different ways depending on where they are used. You can ignore these variations if you always use the root website-relative URL.</span></span>
