# <a name="social-feed-rest-api-reference-for-sharepoint"></a>Социальные канал Справочник по интерфейсу API службы REST для SharePoint
Поиск конечных точек SharePoint REST для чтения и записи в социальных веб-каналов с помощью **SocialRestFeedManager** ресурсов.
Службы SharePoint представлений состояния (REST) позволяет выполнять те же действия, которые можно выполнять с помощью клиентской объектной модели .NET и объектной модели JavaScript. Служба REST предоставляет ресурсы, которые соответствуют объектов SharePoint, свойств и методов. Для использования службы REST, построение и отправлять запросы **POST** и HTTP **GET** конечных точек ресурсов, которые представляют задачи, которые необходимо выполнить.
  
    
    

URI конечных точек для задач, наиболее веб-канала активности начинаются с ресурсом **SocialRestFeedManager** ( `social.feed`), за которым следует  `my` ресурсов или ресурсов `post` :
- Ресурс  `my` представляет текущего пользователя. При использовании встроенных в URI конечной точки контекста запроса задает для текущего пользователя. Например `http://contoso.com/_api/social.feed/my/news` возвращает канала новостей для текущего пользователя.
    
  
- Представляет ресурс  `post` конкретного потока или post. При использовании встроенных в URI конечной точки контекста запроса задает для указанного потока или post. Например `http://contoso.com/_api/social.feed/post/lock` блокирует указанного потока.
    
  
Если в конечную точку ресурса с параметром, метаданные параметров указан в URI или в тексте запроса. По умолчанию службы REST возвращает ответов, отформатированные в протоколе Atom, но можно запросить формат JSON с помощью **Accept** заголовки HTTP. В разделе примеры запросов на полный [Пример запросов REST для задач, веб-канала активности](social-feed-rest-api-reference-for-sharepoint.md#bk_exampleRequests) .

## <a name="my"></a>Мои
<a name="bk_my"> </a>

Получает сведения о текущем пользователе.
  
    
    
Конечная точка **my** устанавливает текущего пользователя в качестве контекста для любой последующих ресурсов в URI. Например `http://contoso.com/_api/social.feed/my/news` возвращает канала новостей для текущего пользователя.
  
    
    

### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/my`
  
    
    

### <a name="request-parameter"></a>Параметр запроса

Нет.
  
    
    

### <a name="response"></a>Ответ

Тип:  [SP. Social.SocialRestActor](social-feed-rest-api-reference-for-sharepoint.md#bk_SocialRestActor)
  
    
    
сведения о текущем пользователе.
  
    
    
Свойства **SocialRestActor** можно вызвать по отдельности в URI, например `http://<siteCollection>/<site>/_api/social.feed/my/me` возвращает только свойство **Me**.

## <a name="myfeedpost"></a>Мои/канал/Post
<a name="bk_myFeedPost"> </a>

Создает корневой post в канал для текущего пользователя.
  
    
    
Вы можете разместить только в контексте текущего пользователя. Не удается создать корневой post в веб-канал другого пользователя, но можно ответить на post другому пользователю. В разделе  [Поместить/ответ](social-feed-rest-api-reference-for-sharepoint.md#bk_postReply).
  
> [!NOTE]
> [!Примечание] Не следует путать этот ресурс  `Post` с ресурсом `Post` , представляющий конкретный поток или post.
  
    
    


### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

## <a name="myfeed"></a>Мой канал /
<a name="bk_myFeed"> </a>

Получает веб-канал активности текущим пользователем ( **Personal** канал типа).
  
    
    

### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/feed`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/feed(MaxThreadCount=10,SortOrder=1,NewerThan=@v)?@v=datetime'2013-01-01T08:00'`

максимальное число потоков, диапазон даты и времени и порядок сортировки. Можно указать любое сочетание следующих свойств, например, можно указать только свойство **MaxThreadCount**.
  
    
    
Псевдоним **@** можно использовать для передачи специальные символы. Например `<siteUri>/_api/social.feed/my/feed(OlderThan=@v)?@v=datetime'2013-01-01T08:00'` используется псевдоним **@v** для отправки **:** символов.

## <a name="mynews"></a>Мои / новостей
<a name="bk_myNews"> </a>

Получает веб-канал активности текущим пользователем и людей и содержимое, которое пользователь следующие, отсортированные по дату последнего изменения ( **News** канал типа).
  
    
    

### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/news`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/news(MaxThreadCount=10,NewerThan=@v)?@v=datetime'2013-01-01T08:00'`
  
    
    

### <a name="request-parameter"></a>Параметр запроса

 _feedOptions_ (необязательно)
  
    
    
Тип:  [SP. Social.SocialFeedOptions](social-feed-rest-api-reference-for-sharepoint.md#bk_SocialFeedOptions)
  
    
    
максимальное число потоков, диапазон даты и времени и порядок сортировки. Можно указать любое сочетание следующих свойств, например, можно указать только свойство **MaxThreadCount**.
  
    
    
Псевдоним **@** можно использовать для передачи специальные символы. Например `<siteUri>/_api/social.feed/my/News(OlderThan=@v)?@v=datetime'2013-01-01T08:00'` используется псевдоним **@v** для отправки **:** символов.

## <a name="mylikes"></a>Мои аудио- и оценки
<a name="bk_myLikes"> </a>

Получает веб-канал микроблога записи в блогах, текущего пользователя мне нравится, представленное типы потоков **LikeReference** . В разделе [Справочник по потоки и потоки дайджест в веб-каналов социальных SharePoint](reference-threads-and-digest-threads-in-sharepoint-server-social-feeds.md).
  
    
    

### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/likes`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/my/likes(MaxThreadCount=10,SortOrder=1,NewerThan=@v)?@v=datetime'2013-01-01T08:00'`
  
    
    

### <a name="request-parameter"></a>Параметр запроса

максимальное число потоков, диапазон даты и времени и порядок сортировки. Можно указать любое сочетание следующих свойств, например, можно указать только свойство **MaxThreadCount**.

## <a name="actor"></a>Actor (субъект)
<a name="bk_actor"> </a>

Получает сведения о указанного пользователя и текущего пользователя.
  
> [!NOTE]
> [!Примечание] Конечная точка  `actor` устанавливает указанного пользователя или сайта веб-канала в качестве контекста для любой последующих ресурсов в URI. Например `http://contoso.com/_api/social.feed/actor(item='domain\\user')/feed` получает личные веб-канала для указанного пользователя и `http://contoso.com/_api/social.feed/actor(item=@v)/feed?@v='http://<server>/<teamSite>/newsfeed.aspx'` получает веб-канал сайта team указанного сайта.
  
    
    


### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item='domain\\user')`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item=@v)?@v='i:0"%23".f|membership|user@domain.com'`
  
    
    

### <a name="request-parameter"></a>Параметр запроса

 _item_
  
    
    
тип: **String**
  
    
    
имя учетной записи указанного пользователя.
  
    
    
Параметр  _item_ отправить в строке запроса. Псевдоним **@** можно использовать для передачи специальные символы. К примеру `<siteUri>/_api/social.feed/actor(item=@v)?@v='i:0"%23".f|membership|user@domain.com'` использует псевдоним **@v** и **"%23"** кодирования для отправки **#** символов.

## <a name="actorfeed"></a>Actor/веб-канал
<a name="bk_actorFeed"> </a>

Получает веб-канал активности по указанному пользователю ( **Personal** канал типа) или получает указанного веб-канал сайта.
  
    
    

### <a name="endpoint-uri-structure"></a>Структура URI конечной точки

 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item='domain\\user')/feed`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item=@v)/feed?@v='i:0"%23".f|membership|user@domain.com'`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item='domain\\user')/feed(MaxThreadCount=10,SortOrder=1,NewerThan=@v)?@v=datetime'2013-01-01T08:00'`
  
    
    
 **GET** `http://<siteCollection>/<site>/_api/social.feed/actor(item=@v)/feed?@v='http://<teamSiteUri>/newsfeed.aspx'`
  
    
    

### <a name="request-parameter"></a>Параметр запроса

 _feedOptions_ (необязательно)

максимальное число потоков, диапазон даты и времени и порядок сортировки. Можно указать любое сочетание следующих свойств, например, можно указать только свойство **MaxThreadCount**.
  
    
    
Псевдоним **@** можно использовать для передачи специальные символы. К примеру `<siteUri>/_api/social.feed/actor(item=@v)/feed(NewerThan=@x)?@v='i:0"%23".f|membership|user@domain.com'&amp;@x=datetime'2013-01-01T08:00'` использует псевдоним **@v** и **"%23"** кодировку для отправки **#** символов и псевдоним **@x** для отправки **:** символов.

## <a name="actorfeedpost"></a>Actor/канал/Post

## <a name="example-rest-requests-for-feed-tasks"></a>Пример запросов REST для задач, веб-канала активности
<a name="bk_exampleRequests"> </a>

 **GET** -запросов для веб-канала активности задачи укажите параметры в URI или в качестве атрибута **url** запроса. запросы на **POST** укажите параметры в атрибуте **data** текст запроса в формате XML или Нотация объектов JavaScript (JSON). Можно сделать HTTP-запросов на любом языке, включая JavaScript и C#. Следующие запросы на примере показано, как мог выполнять запросы с помощью JavaScript и порядок передачи данных сущностей в формате JSON.
  
    
    
 **Пример:** Как задать параметр _ID_ в тексте запроса (в атрибуте **data** ).
  
> [!NOTE]
> [!Примечание] Значения свойств **Id** потока и запись: слишком много времени для отправки в URL-адрес, поэтому должны отправлять их в тексте запроса. В результате даже только операции чтения, логически **GET** запросов необходимо отправить как **POST** запросы. Например для получения потока, необходимо отправить запрос **POST** и передайте поток **Id** как сущность в тексте запроса.
  
    
    




```

var endpoint = siteUrl + '/_api/social.feed/post';
var postId = '1.655c70c348374d48839daabc24a360f0.82baa3bdfa2f481a8185802eb9c6c6cd.5d227a6fa3894f0c9dde26876b71d619.0c37852b34d0418e91c62ac25af4be5b.4316bdaa94bf4984be5dfea1ba96954e.26.26.S-1-5-21-2127521184-1604012920-1887927527-66602';

$.ajax({
    url: endpoint,
    type: 'POST',
    data: JSON.stringify({
        'ID': postId
    }),
    headers: {
        "accept": "application/json;odata=verbose",
        "content-type": "application/json;odata=verbose",
        "X-RequestDigest": $("#__REQUESTDIGEST").val()
    },
    success: function(data) { 
        var stringData = JSON.stringify(data);
        alert(stringData);

        // Converts the response data into an object that you can work with.
        var jsonObject = JSON.parse(stringData);
    },
    error: function(xhr, ajaxOptions, thrownError) {
        alert("Error: " + xhr.status + " " + thrownError + "\\nResponseText: " + xhr.responseText);
    }
});
```

 **Пример:** Как опубликовать запись корневой и указать параметр _restCreationData_ в атрибуте **data**.
  
    
    



```

var endpoint = <site url> + '/_api/social.feed/my/feed/post';
```

<a name="bk_SocialFeedOptions"> </a>

|**Вариант**|**Тип**|**Описание**|

|SortOrder|**Int32**|Порядок сортировки потоков в веб-канал. |:-----|:-----|:-----|
|Attachment| [SP. Social.SocialAttachment](http://msdn.microsoft.com/library/dfdee790-a1b0-19c8-0e92-5a6e058ba4db%28Office.15%29.aspx)|Изображение, видео или вложение документов для post.|

|ContentText|**String**|Обычный текст post, которые могут включать маркеры позиционные вставки (например, "сегодня  {0} день рождения!").|

### <a name="spsocialsocialrestthread"></a>SP. Social.SocialRestThread
<a name="bk_SocialRestThread"> </a>

Представляет поток, который содержит корневой post и набор ответов.
  
    
    
Сервер возвращает **SocialRestThread** ресурсов в ответ на запрос со стороны клиента для создания post или Получение полного потока.
  
    
    
 **SocialRestThread** содержит свойство **ID** и оболочку [SP. Social.SocialThread](http://msdn.microsoft.com/library/46aa4beb-d708-f20e-471e-626c8a7efab7%28Office.15%29.aspx) объекта.
  
    
    

|**Свойство**|**Тип**|**Описание**|
|:-----|:-----|:-----|
|ID|**String**|Уникальный идентификатор потока.|

|**Свойство**|**Тип**|**Описание**|
|:-----|:-----|:-----|
|Субъекты| [SP. Social.SocialActor](http://msdn.microsoft.com/library/4e369fd5-b9b0-9804-957e-b3e39c559cd4%28Office.15%29.aspx)]|Объединенные массива все вовлеченные субъекты.|
|Атрибуты|**Int32**|Побитовое значение, которое представляет набор атрибутов для потока. Просмотреть  [SP. Social.SocialThreadAttributes](http://msdn.microsoft.com/library/21ff9f92-3223-bcc7-ceec-7b899ee29b6e%28Office.15%29.aspx).|
|Id|**String**|Уникальный идентификатор потока.|
|OwnerIndex|**Int32**|Индекс владелец потока в рамках субъекты потока.|
